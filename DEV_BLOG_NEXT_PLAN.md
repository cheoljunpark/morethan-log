# Dev Blog Next Plan

## 목표

이 블로그를 단순한 노션 기반 기록장이 아니라,
읽기 경험, 탐색 경험, 운영 구조, 검색 신뢰도까지 갖춘 개발 블로그로 고도화한다.

핵심 방향은 아래 5가지다.

- 긴 기술 글도 편하게 읽히는 구조
- 글이 많아져도 찾기 쉬운 탐색 구조
- 노션 기반 운영의 제어권 강화
- 개발 블로그다운 신뢰 요소 강화
- 개인 브랜딩과 포트폴리오 역할 확장

## 현재 상태

이미 반영된 것:

- 노션 응답 호환성 보강
- reading time
- previous / next navigation
- related posts
- series 지원
- 검색 / 카테고리 / 정렬 상태 유지
- 기본 SEO / canonical / JSON-LD 보강
- 홈 피드 카드 UI 개선

현재 부족한 것:

- 아카이브성 탐색 구조 부족
- 홈에서 핵심 글을 큐레이션하는 힘 부족
- 노션 속성 기반 운영 기능 부족
- 글 신뢰도를 높이는 메타 정보 부족
- 장기적으로 글이 많아졌을 때 탐색성이 약함

## 1. 가장 먼저 필요한 고도화

### 1. Pinned / Featured 운영 시스템

목표:
운영자가 “보여주고 싶은 글”을 홈에서 직접 통제할 수 있게 만든다.

필요 작업:

- `pinned` 노션 속성 지원
- `featured` 노션 속성 지원
- 홈 상단 `Featured Writing` / `Start Here` 섹션 분리
- 카테고리별 대표 글 1개 노출

효과:

- 홈 완성도 상승
- 새 방문자 유입 시 첫인상 개선
- 운영 편의성 상승

### 2. Archive / Tags / Series 탐색 페이지

목표:
글이 쌓일수록 블로그 가치가 올라가게 만든다.

필요 작업:

- `/archive`
- `/tags`
- `/series`
- 연도별 / 태그별 / 시리즈별 묶음 페이지

효과:

- 오래된 글이 묻히지 않음
- 검색 외 탐색 경로 강화
- 개발 블로그다운 자료성 상승

### 3. 글 상단 요약 박스

목표:
긴 기술 글을 더 빠르게 이해하게 만든다.

형식 예시:

- 이 글에서 다루는 것
- 문제
- 원인
- 해결
- 배운 점

효과:

- 검색 유입 사용자 체류 시간 증가
- 글 품질 인상 상승

## 2. 심화 고도화

### 4. 신뢰도 메타 강화

목표:
기술 글로서 “정리된 글”이라는 인상을 강하게 만든다.

필요 작업:

- `updatedAt` 안정 노출
- `last reviewed` 개념 추가
- 참고 링크 / 레퍼런스 섹션 추가
- GitHub / demo / docs 링크 블록 추가
- heading anchor copy

효과:

- 블로그 신뢰도 상승
- 기술 문서에 가까운 완성도 확보

### 5. 콘텐츠 모델 심화

추천 노션 속성:

- `pinned`
- `featured`
- `updatedAt`
- `difficulty`
- `series`
- `seriesOrder`
- `hideFromHome`
- `hideFromSitemap`
- `canonicalUrl`
- `externalLink`

효과:

- 배포 없이 홈 구성 조정 가능
- SEO 제어 가능
- 시리즈 운영 가능

### 6. 검색 경험 강화

목표:
검색이 “필터”가 아니라 실제 탐색 도구가 되게 만든다.

필요 작업:

- 검색어 debounce
- 검색 결과 강조 표시
- 제목 / 요약 / 태그 / 시리즈 별 가중치 적용
- 검색어 추천
- 결과 없음 상태 UX 개선

효과:

- 글 수가 많아질수록 유용해짐
- 사용자가 원하는 글을 더 빨리 찾음

### 7. 글 상세 품질 강화

필요 작업:

- 코드 블록 line highlight
- 특정 heading deep-link 복사
- 참고자료 / 관련 문서 섹션
- 이 글을 읽기 전에 보면 좋은 글
- 이 글 다음에 읽으면 좋은 글

효과:

- 글 간 연결성이 강해짐
- 개발 블로그다운 큐레이션 강화

## 3. 중장기 고도화

### 8. About / Projects / Uses 확장

목표:
블로그와 포트폴리오 역할을 같이 수행한다.

필요 작업:

- `/about` 확장
- `/projects` 추가
- `/uses` 추가
- 커리어 타임라인
- 발표 / 글 / 활동 정리

### 9. Analytics 기반 운영

필요 작업:

- 인기 글
- 검색어 집계
- 유입별 클릭 추적
- 이탈률 높은 글 파악

효과:

- 감이 아니라 데이터로 운영 가능

### 10. 퍼블리싱 자동화

가능한 방향:

- 글 작성 체크리스트 자동 검증
- slug 중복 검사
- summary 길이 검사
- 공개 전 필수 속성 검증
- broken internal link 검사

효과:

- 운영 실수 감소
- 품질 유지

## 추천 우선순위

가장 효율적인 순서:

1. `pinned` / `featured` 노션 속성 지원
2. `/archive` 페이지 추가
3. 글 상단 요약 박스 추가
4. `updatedAt` / 참고 링크 / anchor copy
5. `/tags` / `/series` 탐색 페이지 추가

## 구현안 제안

### A안. 운영 중심

먼저 할 것:

- pinned
- featured
- updatedAt
- hideFromHome

추천 상황:

- 홈 구성과 노출을 운영자가 직접 조절하고 싶을 때

### B안. 독자 경험 중심

먼저 할 것:

- archive
- tags
- summary box
- related navigation 강화

추천 상황:

- 검색 유입 독자의 읽기 경험을 먼저 올리고 싶을 때

### C안. 브랜딩 중심

먼저 할 것:

- about
- projects
- 대표 글 큐레이션
- 프로필 / 블로그 설명 정리

추천 상황:

- 블로그 + 포트폴리오를 같이 강화하고 싶을 때
